A partir d‚Äôici, beaucoup de personnes copient les coefficients dans leur traitement de texte. Avec un ensemble de frictions :

  • L‚Äôintervalle de confiance n‚Äôest pas directement disponible lorsque l‚Äôon explore des donn√©es3.
  • Il n‚Äôest pas possible d‚Äôobtenir un coefficient standardis√© type \(\beta\) sans r√©-ajuster le mod√®le. Ici, ce n‚Äôest peut-√™tre pas n√©cessaire car les quantit√©s estim√©es ont un sens. Mais en psychologie‚Ä¶

lm_summary <- function(model, vcov = NULL) {
  
  # Extraire les param√®tres non standardis√©s
  mp <- parameters::model_parameters(model,
                                     vcov = vcov)
  
  # Extraire les param√®tres standardis√©s (refit le mod√®le)
  mp_std <- dplyr::select(
    parameters::model_parameters(
      model,
      standardize = "refit",
      vcov = vcov),
    
    Parameter,
    Std_Coefficient = Coefficient,
    Std_CI_low = CI_low,
    Std_CI_high = CI_high)
  
  mp <- dplyr::left_join(mp, mp_std, by = "Parameter")
  
  # Pr√©parer les donn√©es
  mp <- dplyr::mutate(mp,
                      
                      # Arrondir p √† trois d√©cimales
                      p = ifelse(p < .001, "< .001", round(p, 3)),
                      
                      # Arrondir les autres valeurs √† deux d√©cimales
                      dplyr::across(dplyr::where(is.numeric), ~ round(.x, 2)),
                      
                      # R√©unir l'intervalle de confiance de l'estimate et le placer sous le Coefficient
                      Coefficient = paste0(Coefficient, "<br>", "[", CI_low, "; ", CI_high, "]"),
                      
                      # R√©unir l'intervalle de confiance de Œ≤ et le placer sous Œ≤
                      Œ≤ = paste0(Std_Coefficient, "<br>", "[", Std_CI_low, "; ", Std_CI_high, "]"),
                      
                      # Conserver le mod√®le pour lm_tab, m√™me apr√®s des verbes dplyr (mutate, filter, etc.)
                      .model = list(model))
  
  # Renommer les colonnes : les colonnes avec un nom simple
  mp <- dplyr::rename(mp,
                      Param√®tre = Parameter,
                      Estimation = Coefficient,
                      "Erreur standard" = SE,
                      p = p)
  
  # Renommer les colonnes : la colonne t est un peu plus complexe, car je souhaite y int√©grer le nombre de degr√©s de libert√© de la statistique
  mp <- dplyr::rename_with(mp,
                           ~ paste0("Statistique t (", broom::glance(model)$df.residual, ")"),
                           .cols = t)
  
  # Ne pas garder les colonnes qui ont d√©j√† √©t√© incorpor√©es √† d'autres, ou ne sont pas utiles
  mp <- dplyr::select(mp,
                      -c(CI, CI_low, CI_high, df_error, Std_Coefficient, Std_CI_low, Std_CI_high))
  
  mp
}

lm_tab <- function(lm_summary) {
  
  lm_model <- attr(lm_summary, "model")
  if (is.null(lm_model) && ".model" %in% names(lm_summary)) {
    lm_model <- lm_summary$.model[[1]]
  }
  if (is.null(lm_model)) {
    stop("Aucun mod√®le retrouv√© en sortie de lm_summary.")
  }
  
  g <- broom::glance(lm_model)
  t_col <- paste0("Statistique t (", g$df.residual, ")")
  
  tab <- gt::gt(lm_summary)
  
  # Ne pas afficher la colonne technique, qui contient le mod√®le
  tab <- gt::cols_hide(tab, columns = dplyr::any_of(".model"))
  
  tab <- gt::cols_label(
    tab,
    Param√®tre = gt::html("<strong>Param√®tre</strong>"),
    Estimation = gt::html("<strong>Estimation<br>[IC 95 %]</strong>"),
    `Erreur standard` = gt::html("<strong>Erreur standard</strong>"),
    !!t_col := gt::html(
      paste0("<strong>Statistique <em>t</em> (", g$df.residual, ")</strong>")),
    p = gt::html("<strong><em>p</em>-valeur</strong>"),
    Œ≤ = gt::html("<strong><em>Œ≤</em><br>[IC 95 %]</strong>"))
  
  tab <- gt::fmt_markdown(tab, columns = c(Estimation, Œ≤))
  
  tab <- gt::cols_align(
    tab,
    align = "center",
    columns = c(Estimation, `Erreur standard`, dplyr::all_of(t_col), p, Œ≤))
  
  tab <- gt::tab_source_note(
    tab,
    gt::html(paste0("<em>Note</em> : n = ",
                    scales::number(g$nobs),
                    " observations.")))
  
  tab
}

A partir d'ici, beaucoup de personnes copient les coefficients dans leur traitement de texte. Avec un ensemble de frictions : 

-   Si le mod√®le change, il faut re-copier les nouvelles estimations. 
-   Si l'on copie manuellement, des erreurs peuvent appara√Ætre. 
-   L'intervalle de confiance n'est pas directement disponible lorsque l'on explore des donn√©es^[Peut-√™tre que vous arrivez √† calculer `r round(lm_1$coefficients[1], 2)` + (1.96 x `r round(summary(lm_1)$coefficients[1, 2], 2)`) de t√™te, personnellement, je ne sais pas faire.]. 
-   Il n'est pas possible d'obtenir un coefficient standardis√© type $\beta$ sans r√©-ajuster le mod√®le. Ici, ce n'est peut-√™tre pas n√©cessaire car les quantit√©s estim√©es ont un sens. Mais en psychologie... 

Bref : on peut faire mieux^[Et d'excellents packages existent pour extraire de magnifiques tables de r√©gression, dont l'excellent `modelsummary` [@modelsummary] ou plus anciennement `stargazer` qui n'est aujourd'hui plus mis √† jour [@stargazer].]. J'ai donc r√©dig√© une petite fonction, e que s'apelerio `lm_summary()`, pour extraire les r√©sultats d'un mod√®le `lm`. Cette fonction prend deux arguments : un mod√®le lin√©aire `lm` (duh!) et un argument `vcov` pour sp√©cifier la matrice de variance-covariance √† utiliser. Elle s'appuie sur le package `parameters` [@parameters] pour extraire les donn√©es du mod√®le lin√©aire, puis sur `dplyr` [@dplyr] pour structurer la table.

```{r}
#| label: tbl-regression-lm_summary
#| tbl-cap: "R√©gression de la masse des manchots en grammes, sur l'esp√®ce et le sexe, selon l'@eq-un. Sortie de la fonction `lm_summary()` pass√©e via `knitr::kable()`."

lm_summary <- function(model, vcov = NULL) {
  
  # Extraire les param√®tres non standardis√©s
  mp <- parameters::model_parameters(model,
                                     vcov = vcov)
  
  # Extraire les param√®tres standardis√©s (refit le mod√®le)
  mp_std <- dplyr::select(
    parameters::model_parameters(
      model,
      standardize = "refit",
      vcov = vcov),
    
    Parameter,
    Std_Coefficient = Coefficient,
    Std_CI_low = CI_low,
    Std_CI_high = CI_high)
  
  mp <- dplyr::left_join(mp, mp_std, by = "Parameter")
  
  # Pr√©parer les donn√©es
  mp <- dplyr::mutate(mp,
                      
                      # Arrondir p √† trois d√©cimales
                      p = ifelse(p < .001, "< .001", round(p, 3)),
                      
                      # Arrondir les autres valeurs √† deux d√©cimales
                      dplyr::across(dplyr::where(is.numeric), ~ round(.x, 2)),
                      
                      # R√©unir l'intervalle de confiance de l'estimate et le placer sous le Coefficient
                      Coefficient = paste0(Coefficient, "<br>", "[", CI_low, "; ", CI_high, "]"),
                      
                      # R√©unir l'intervalle de confiance de Œ≤ et le placer sous Œ≤
                      Œ≤ = paste0(Std_Coefficient, "<br>", "[", Std_CI_low, "; ", Std_CI_high, "]"),
                      
                      # Conserver le mod√®le pour lm_tab, m√™me apr√®s des verbes dplyr (mutate, filter, etc.)
                      .model = list(model))
  
  # Renommer les colonnes : les colonnes avec un nom simple
  mp <- dplyr::rename(mp,
                      Param√®tre = Parameter,
                      Estimation = Coefficient,
                      "Erreur standard" = SE,
                      p = p)
  
  # Renommer les colonnes : la colonne t est un peu plus complexe, car je souhaite y int√©grer le nombre de degr√©s de libert√© de la statistique
  mp <- dplyr::rename_with(mp,
                           ~ paste0("Statistique t (", broom::glance(model)$df.residual, ")"),
                           .cols = t)
  
  # Ne pas garder les colonnes qui ont d√©j√† √©t√© incorpor√©es √† d'autres, ou ne sont pas utiles
  mp <- dplyr::select(mp,
                      -c(CI, CI_low, CI_high, df_error, Std_Coefficient, Std_CI_low, Std_CI_high))
  
  mp
}

lm_summary(lm_1) %>%
  select(-c(.model)) %>% # Une colonne ".model" stocke le mod√®le, pour pouvoir l'envoyer dans une future fonction. C'est in√©l√©gant, mais √ßa fonctionne.
  knitr::kable()
```

Ce dataframe est un bon d√©but √† mes yeux, m√™me s'il a des limites. Je combine cette fonction avec une seconde, qui va mettre cette table en forme. Celle-ci, `lm_tab()`, s'appuie principalement sur le package `gt` [@gt].

```{r}
#| label: tbl-regression-lm_tab
#| tbl-cap: "R√©gression de la masse des manchots en grammes, sur l'esp√®ce et le sexe, selon l'@eq-un. Sortie de la fonction `lm_summary()` pass√©e via `lm_tab()`."

lm_tab <- function(lm_summary) {
  
  lm_model <- attr(lm_summary, "model")
  if (is.null(lm_model) && ".model" %in% names(lm_summary)) {
    lm_model <- lm_summary$.model[[1]]
  }
  if (is.null(lm_model)) {
    stop("Aucun mod√®le retrouv√© en sortie de lm_summary.")
  }
  
  g <- broom::glance(lm_model)
  t_col <- paste0("Statistique t (", g$df.residual, ")")
  
  tab <- gt::gt(lm_summary)
  
  # Ne pas afficher la colonne technique, qui contient le mod√®le
  tab <- gt::cols_hide(tab, columns = dplyr::any_of(".model"))
  
  tab <- gt::cols_label(
    tab,
    Param√®tre = gt::html("<strong>Param√®tre</strong>"),
    Estimation = gt::html("<strong>Estimation<br>[IC 95 %]</strong>"),
    `Erreur standard` = gt::html("<strong>Erreur standard</strong>"),
    !!t_col := gt::html(
      paste0("<strong>Statistique <em>t</em> (", g$df.residual, ")</strong>")),
    p = gt::html("<strong><em>p</em>-valeur</strong>"),
    Œ≤ = gt::html("<strong><em>Œ≤</em><br>[IC 95 %]</strong>"))
  
  tab <- gt::fmt_markdown(tab, columns = c(Estimation, Œ≤))
  
  tab <- gt::cols_align(
    tab,
    align = "center",
    columns = c(Estimation, `Erreur standard`, dplyr::all_of(t_col), p, Œ≤))
  
  tab <- gt::tab_source_note(
    tab,
    gt::html(paste0("<em>Note</em> : n = ",
                    scales::number(g$nobs),
                    " observations.")))
  
  tab
}

lm_1 %>%
  lm_summary() %>%
  mutate(
    Param√®tre = case_when(
      Param√®tre == "especeJugulaire" ~ "Esp√®ce : Jugulaire",
      Param√®tre == "especePapou" ~ "Esp√®ce : Papou",
      Param√®tre == "sexeM√¢le" ~ "Sexe : M√¢le",
      .default = Param√®tre)) %>%
  lm_tab()
```

Ces fonctions ne me plaisent aujourd'hui qu'√† moiti√©, car elles n√©cessitent de modifier manuellement le dataframe en sortie de `lm_summary()` pour renommer les param√®tres avant de l'envoyer √† `lm_tab()`. Si le dataframe est modifi√© en amont de l'ajustement du mod√®le, le pipe peut casser et la fonction ne plus fonctionner. 

Mais, √† d√©faut, cela permet malgr√© tout d'obtenir une table qui synth√©tise toutes les informations n√©cessaires de fa√ßon √©l√©gante. De plus, cette table √©tant un objet `gt`, elle peut √™tre export√©e via la fonction `gtsave()` pour √©viter de s'emb√™ter √† copier-coller manuellement les r√©sultats.

Il reste deux probl√®mes √† mon sens : 

-   En interne, tab_model() a besoin de la colonne de l'erreur standard de $\beta$ (`standardized std. Error`) pour calculer l'intervalle de confiance. Mais, je ne souhaite pas afficher cette erreur. -   Par d√©faut, le style de table affiche les noms de colonnes en italique, et je n'arrive pas √† ce que cela ne soit pas le cas.
